1. 1 两数之和

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

示例 1：

输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。

3. 128 最长连续序列

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9

原本我是这么写的

但是我忽略了min和max函数遍历数组带来的内存过大问题

至于改进上：

  • 使用集合：可以使用集合 set 来快速查找数字是否存在，而不需要维护索引。
  • 避免遍历整个范围：只需要遍历 nums 中实际出现的数字，不需要遍历从 min(nums) 到 max(nums) 的所有整数。
  • 避免重复检查：在遍历过程中，只需要从序列中的起始点开始找一次连续序列，避免重复计算。

这样遍历集合中的每个元素时，最坏情况下每个元素会被检查一次，总的时间复杂度是 O(n)。

6. 15 三数之和

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。

输入: s = “cbaebabacd”, p = “abc”
输出: [0,6]
解释:
起始索引等于 0 的子串是 “cba”, 它是 “abc” 的异位词。
起始索引等于 6 的子串是 “bac”, 它是 “abc” 的异位词。

起始索引等于 0 的子串是 “ab”, 它是 “ab” 的异位词。
起始索引等于 1 的子串是 “ba”, 它是 “ab” 的异位词。
起始索引等于 2 的子串是 “ab”, 它是 “ab” 的异位词。

10. 560 和为k的子数组

给你一个整数数组 nums 和一个整数 k ，请你统计并返回 该数组中和为 k 的子数组的个数 。

子数组是数组中元素的连续非空序列。

输入：nums = [1,1,1], k = 2
输出：2

输入：nums = [1,2,3], k = 3
输出：2

-1000 <= nums[i] <= 1000

一边计算前缀和，一边遍历前缀和。

或者是两次遍历

对于上面的 ans += cnt[sj - k]

  • 如果存在一个前缀和 si，使得 sj - si = k，则子数组 nums[i:j] 的和等于 k。
  • 通过哈希表 cnt，快速查询是否存在这样的 si，即检查 sj−k 是否在 cnt 中。
  • 如果 sj−k在 cnt 中，说明从某个之前位置 i 开始到当前的子数组和为 k，将其出现次数 cnt[sj - k] 加到答案 ans 中。

这个题不能用滑动窗口

滑动窗口需要满足单调性，当右端点元素进入窗口时，窗口元素和是不能减少的。本题 nums 包含负数，当负数进入窗口时，窗口左端点反而要向左移动，导致算法复杂度不是线性的。

数组不是单调的话，不要用滑动窗口，考虑用前缀和
